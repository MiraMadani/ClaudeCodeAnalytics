
# LLM Usage

## AI-Assisted Development

This project was developed using AI-assisted development with **OpenAI Codex CLI**.

The AI agent was used to accelerate implementation, improve code quality, and automate repetitive development tasks. Every generated component was reviewed, tested, and integrated manually.

---

## AI-Generated Components

The agent assisted in generating or improving:

- project structure;
- ETL pipeline;
- data ingestion;
- data cleaning;
- validation helpers;
- DuckDB schema;
- repository layer;
- analytics SQL queries;
- Dash dashboard;
- Plotly visualizations;
- documentation.

---

## Development Process

The development workflow consisted of the following steps:

1. Analyze the task requirements.
2. Generate an implementation proposal using Codex CLI.
3. Review the generated solution.
4. Adapt the implementation when necessary.
5. Test the feature locally.
6. Integrate it into the project.

This iterative process was repeated throughout the project.

---

## Human Contribution

The project owner was responsible for:

- understanding the generated code;
- making architectural decisions;
- integrating all project components;
- debugging runtime errors;
- validating business logic;
- testing the ETL pipeline and dashboard;
- organizing the repository;
- preparing documentation.

---

## Challenges Solved

During development several issues were resolved, including:

- flattening nested telemetry JSON structures;
- designing a normalized DuckDB schema;
- handling malformed telemetry records;
- implementing data validation;
- connecting analytics queries with the dashboard;
- debugging Dash callbacks;
- configuring Git and GitHub;
- excluding large datasets from the repository.

---

## Final Result

AI significantly accelerated development by automating repetitive coding tasks, while all architectural decisions, testing, debugging, and final integration remained under human supervision.
